# Vim


## What is Vim?

`vim` (Vi Improved) is a powerful, lightweight **terminal-based text editor**.

Unlike normal editors, Vim is **mode-based**, meaning different keys perform different actions depending on the current mode.



## Why Vim is useful in Cloud and DevOps

### 1. Available Everywhere

* Pre-installed on most Linux servers
* No need to install GUI editors
* Works over SSH



### 2. Fast and Efficient

* Keyboard-driven (no mouse required)
* Very fast for editing configs and code
* Ideal for quick fixes on servers



### 3. Essential for Configuration Work

Common DevOps tasks:

* Editing YAML files (Kubernetes)
* Updating config files (`.conf`, `.env`)
* Writing scripts



### 4. Lightweight

* Runs in minimal environments
* Works inside containers, VMs, remote servers



## Core Concept: Modes in Vim

Vim has three main modes:

```
Normal Mode   → Navigation and commands
Insert Mode   → Typing text
Command Mode  → Saving, quitting, operations
```


---
## Starting Vim

```
vim file.txt
```

If file does not exist, it will be created.



## Basic Usage

### Enter Insert Mode (start typing)

```
i
```



### Exit Insert Mode

```
Esc
```



### Save file

```
:write
```

or

```
:w
```



### Quit

```
:quit
```

or

```
:q
```



### Save and Quit

```
:wq
```



### Force Quit (discard changes)

```
:q!
```



## Navigation (Normal Mode)

Move cursor:

```
h → left
l → right
k → up
j → down
```



### Jump faster

```
gg → top of file
G  → bottom of file
```



## Editing Basics

### Delete

```
dd → delete line
```



### Copy (yank)

```
yy → copy line
```



### Paste

```
p → paste below
```



### Undo / Redo

```
u     → undo
Ctrl+r → redo
```



## Search

```
/word
```

Press `n` to go to next match.

---

## DevOps Example

Editing a Kubernetes YAML file:

```
vim deployment.yaml
```

Steps:

1. Press `i` → enter insert mode
2. Edit image/version/config
3. Press `Esc`
4. Save:

```
:wq
```



## Real Scenario

### Without Vim

* No GUI editor on server
* Cannot easily edit files



### With Vim

```
ssh server
vim config.yaml
```

* Edit directly on server
* Save and apply changes instantly



## Mental Model

```
Open file → Navigate (Normal Mode)
         → Edit (Insert Mode)
         → Save/Quit (Command Mode)
```



## When to Use Vim

Use Vim when you:

* Work on remote servers
* Edit configs frequently
* Need fast terminal editing
* Work in DevOps / cloud environments



## When Not Necessary

* Heavy coding with IDE features
* GUI-based development workflows
* Beginners uncomfortable with keyboard shortcuts (initially)



## Summary

* `vim` is a powerful terminal editor
* Works everywhere in cloud environments
* Fast, lightweight, and efficient
* Essential for DevOps tasks like config editing
