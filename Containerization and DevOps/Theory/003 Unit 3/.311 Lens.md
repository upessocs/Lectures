# 1. What is Lens?

Lens is a **Kubernetes IDE (Graphical Tool)**.

## Why Lens is Needed

Normally, you use:

```bash
kubectl get pods
kubectl describe pod
kubectl logs
```

This is powerful but:

* Hard for beginners to visualize
* No clear relationship between resources
* Debugging is slower

---

## What Lens Provides

Lens gives a **visual interface for Kubernetes**.

You can:

* View Pods, Deployments, Services
* See cluster health
* View logs with one click
* Edit YAML visually
* Monitor CPU / memory
* Debug easily

---

## Simple Comparison

```text
kubectl = command line control
Lens    = visual dashboard + debugger
```

---

## How Lens Works

Lens does NOT create a cluster.

It:

```text
Reads kubeconfig → Connects to cluster → Displays resources
```

So it uses the same config as `kubectl`.

---

# 2. Install Lens (Windows)

Since you're using WSL + k3d, install Lens on **Windows (not WSL)**.

## Steps

1. Go to:
   - [https://k8slens.dev](https://k8slens.dev)
   - [https://docs.k8slens.dev/k8slens/getting-started/install-lens/#install-lens-desktop-from-the-apt-repository](https://docs.k8slens.dev/k8slens/getting-started/install-lens/#install-lens-desktop-from-the-apt-repository)

2. Download:

   * `.exe` or `.dmg` or `.deb`as per your os

3. Install and launch

---

# 3. Connect Lens to Your k3d Cluster

## Step 1: Ensure cluster is running (WSL)

```bash
k3d cluster start mycluster
kubectl get nodes
```

---

## Step 2: Copy kubeconfig to Windows

In WSL:

```bash
mkdir -p /mnt/c/Users/%USERNAME%/.kube
cp ~/.kube/config /mnt/c/Users/%USERNAME%/.kube/config
```

---

## Step 3: Fix Server Address (Important)

Edit in WSL:

```bash
nano ~/.kube/config
```

Find:

```yaml
server: https://0.0.0.0:6443
```

Change to:

```yaml
server: https://127.0.0.1:6443
```

Save and copy again.

---

## Step 4: Open Lens

* Launch Lens
* It will auto-detect cluster OR

Manual:

```text
Add Cluster → Select kubeconfig
```

---

## Step 5: Connect

* Click cluster name
* Click **Connect**

You should now see:

```text
Nodes
Workloads
Networking
Storage
```

---

# 4. Verify Lens is Working (Very Important)

## Check Nodes

In Lens:

* Go to **Nodes**

You should see:

```text
1 node (k3d cluster)
Status: Ready
```

---

## If Not Working

### Issue: Cluster not visible

* Restart Lens
* Re-copy kubeconfig

---

### Issue: Connection error

* Ensure cluster is running:

```bash
k3d cluster start mycluster
```

---

### Issue: TLS / server error

* Fix:

```text
0.0.0.0 → 127.0.0.1
```

# 5. Mini Exercise to Test Lens

Now that Lens is working, run this **very small test**.

---

## Step 1: Create Deployment

```bash
kubectl create deployment web --image=nginx
```

---

## Step 2: Observe in Lens

Go to:

```text
Workloads → Deployments
```

You should see:

```text
web (1/1 pods running)
```

---

## Step 3: Check Pods

Go to:

```text
Workloads → Pods
```

You should see:

```text
1 pod running
```

Click the pod and explore:

* Logs tab
* YAML tab

---

## Step 4: Scale Application

```bash
kubectl scale deployment web --replicas=3
```

---

## Observe in Lens

Pods should become:

```text
3 running pods
```

---

## Step 5: Delete a Pod

```bash
kubectl get pods
kubectl delete pod <pod-name>
```

---

## Observe in Lens

```text
Deleted pod → new pod created automatically
```

This demonstrates:

```text
Kubernetes self-healing
```

---

# 6. What Students Should Understand

After this exercise:

```text
Deployment → manages pods
Pods → running containers
Lens → visualizes everything
kubectl → controls everything
```

# 7. Final Teaching Insight

Best way to teach:

```text
1. Run command (kubectl)
2. Show effect (Lens)
3. Explain concept
```
